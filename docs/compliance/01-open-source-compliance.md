# 开源协议合规方案

> 本文档详细描述项目使用 OpenClaw 框架的开源协议合规方案，确保商业化合法合规。

---

## 1. OpenClaw 许可证分析

### 1.1 OpenClaw 许可证

OpenClaw 采用 **MIT 许可证**，是世界上最开放的开源许可证之一。

### 1.2 MIT 许可证核心条款

```
┌─────────────────────────────────────────────────────────────┐
│                        MIT 许可证条款                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  MIT License                                               │
│                                                             │
│  Copyright (c) [year] [copyright holders]                   │
│                                                             │
│  Permission is hereby granted, free of charge, to any       │
│  person obtaining a copy of this software...                │
│                                                             │
│  Key rights granted:                                       │
│  ├── ✅ 用于任何目的（包括商业用途）                          │
│  ├── ✅ 修改、再分发                                         │
│  ├── ✅ 私有使用                                             │
│  ├── ✅ 销售衍生产品                                         │
│                                                             │
│  Key obligations:                                          │
│  ├── 📝 必须保留原始版权声明                                 │
│  └── 📝 必须在分发时包含许可证全文                           │
│                                                             │
│  Key restrictions:                                         │
│  ├── ❌ 不能声称是官方版本                                   │
│  └── ❌ 不能用原作者名义进行背书                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 与其他许可证对比

| 许可证 | 商业使用 | 修改后必须开源 | 专利授权 | 典型项目 |
|--------|----------|---------------|----------|----------|
| **MIT** | ✅ | ❌ | ❌ | OpenClaw, jQuery, Rails |
| **Apache 2.0** | ✅ | ❌ | ✅ | Hadoop, Android |
| **GPL 3.0** | ✅ | ✅ | ✅ | Linux, Git |
| **AGPL 3.0** | ✅ | ✅ | ✅ | MongoDB |
| **LGPL 3.0** | ✅ | 部分 | ✅ | Qt |

MIT 是最宽松的许可证之一，适合商业化产品。

## 2. 合规要求清单

### 2.1 必须执行的要求

| 要求 | 说明 | 执行方式 |
|------|------|----------|
| **保留版权声明** | 在所有包含 OpenClaw 代码的文件中保留原始版权声明 | 自动/手动检查 |
| **分发许可证** | 分发软件时必须包含 MIT 许可证全文 | 放在产品文档中 |
| **声明使用** | 在产品中声明使用了 OpenClaw | 在"关于"页面说明 |

### 2.2 禁止的行为

| 禁止行为 | 示例 | 风险等级 |
|----------|------|----------|
| **冒用官方名义** | 声称"本产品是 OpenClaw 官方产品" | 高 |
| **虚假宣传** | 暗示得到 OpenClaw 官方支持或认可 | 中 |
| **移除版权声明** | 删除代码中的版权声明 | 高 |

## 3. 合规实施方案

### 3.1 产品文档合规

#### 关于页面内容

```markdown
┌─────────────────────────────────────────────────────────────┐
│                        关于页面                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [产品名称] v1.0.0                                          │
│                                                             │
│  本产品基于 OpenClaw（MIT 许可证）开发。                     │
│                                                             │
│  OpenClaw 版权声明：                                        │
│  Copyright (c) [year] OpenClaw Authors                     │
│  Permission is hereby granted...                           │
│  (完整 MIT 许可证全文)                                       │
│                                                             │
│  许可证原文：https://opensource.org/licenses/MIT           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 用户手册合规

在用户手册的"法律信息"章节包含：
- OpenClaw 版权声明
- MIT 许可证全文
- 开源软件使用清单（如有其他开源组件）

### 3.2 代码层面合规

#### 版权声明保留

在基于 OpenClaw 开发的代码文件中保留原始声明：

```typescript
// 基于 OpenClaw (MIT License) 开发
// Copyright (c) [year] OpenClaw Authors
// 本项目修改部分版权属于 [公司名]
```

#### 许可证文件

在项目根目录或文档目录包含：

```
docs/
├── licenses/
│   ├── MIT-LICENSE.txt          # OpenClaw MIT 许可证全文
│   └── THIRD-PARTY-LICENSES.txt # 其他开源组件许可证
```

### 3.3 分发合规

| 分发场景 | 合规要求 |
|----------|----------|
| **SaaS服务** | 不需要开源，但需要保留版权声明 |
| **客户端软件** | 需要包含许可证文件 |
| **SDK/API** | 需要包含许可证文件 |
| **源代码分发** | 需要保留所有版权声明 |

## 4. 许可证兼容性

### 4.1 本项目的许可证组合

```
┌─────────────────────────────────────────────────────────────┐
│                    许可证层次结构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  商业专有层                                                 │
```

### 4.2 许可证兼容性分析

| 许可证 | 与 MIT 兼容 | 与商业代码结合 | 说明 |
|--------|-------------|---------------|------|
| MIT | ✅ | ✅ | 最宽松 |
| Apache 2.0 | ✅ | ✅ | 需保留声明 |
| BSD | ✅ | ✅ | 类似 MIT |
| GPL 3.0 | ⚠️ | ❌ | 需开源 |
| AGPL 3.0 | ⚠️ | ❌ | 需开源 |

**结论**：本项目选择的依赖均为 MIT/Apache 2.0 等宽松许可证，与商业化完全兼容。

## 6. 常见问题

### 6.1 是否需要开源我的代码？

**不需要**。MIT 许可证不要求衍生产品开源。您可以：
- 完全闭源
- 商业销售
- 私有使用

### 6.2 是否需要支付费用？

**不需要**。MIT 许可证完全免费，不收取任何费用。

### 6.3 是否需要提供技术支持？

**不需要**。MIT 许可证不包含任何担保或支持义务。

### 6.4 可以删除版权声明吗？

**不可以**。必须保留原始版权声明。删除版权声明违反许可证。

### 6.5 可以声称是官方版本吗？

**不可以**。不能声称或暗示本产品是 OpenClaw 的官方版本。

## 8. 结论

OpenClaw 的 MIT 许可证对商业化非常友好：

1. **使用自由**：可用于任何商业目的
2. **无传染性**：无需开源您的代码
3. **要求简单**：仅需保留版权声明
4. **完全兼容**：与商业授权软件无冲突

通过遵循本文档的合规方案，可以确保产品合法合规使用 OpenClaw。
