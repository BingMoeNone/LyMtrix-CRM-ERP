# OpenClaw 内核定制开发方案

> 本文档描述以 OpenClaw 为核心内核，在其基础上进行深度定制开发的架构方案。

---

## 1. 架构定位

### 1.1 核心定位

```
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw 内核定制开发架构                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【我们的定制层】                                           │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  业务层：风险分级层、业务规则层、人工确认层          │  │
│  │  决策层：分层投票层、动态风险层、白名单层            │  │
│  │  接入层：API适配层、协议转换层、展示层              │  │
│  └─────────────────────────────────────────────────────┘  │
│                           ▲                                │
│                           │ 深度定制                        │
│                           │                                │
│  【OpenClaw 内核层】                                    │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Agent生命周期管理 | 工具调用调度 | 会话管理        │  │
│  │  MCP协议支持 | Human-in-the-Loop | 日志审计          │  │
│  └─────────────────────────────────────────────────────┘  │
│                           ▲                                │
│                           │ 使用                           │
│                           │                                │
│  【模型层】MiniMax M2.7 / M2.5                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 开发理念

| 理念 | 说明 |
|------|------|
| **OpenClaw作为内核** | 仅使用其Agent调度、工具调用、会话管理能力 |
| **业务逻辑自主开发** | 风险分级、投票决策、人工确认等核心逻辑自主实现 |
| **不受限于框架** | OpenClaw的能力是基础，业务需求是主导 |
| **可替换性** | 如果OpenClaw不满足需求，可以替换为其他方案 |

## 2. 与"二次开发"的区别

### 2.1 二次开发 vs 内核定制

| 维度 | 二次开发 | 内核定制（我们的方式） |
|------|----------|------------------------|
| **定位** | 在OpenClaw基础上扩展 | 以OpenClaw为底层内核 |
| **业务逻辑** | 尽量用OpenClaw原生能力 | 核心逻辑自主实现 |
| **扩展方式** | 插件化、配置文件 | 独立模块替换 |
| **绑定程度** | 强依赖OpenClaw特性 | 弱依赖，可替换 |
| **开发自由度** | 受OpenClaw约束 | 完全自主 |

### 2.2 我们不使用的OpenClaw原生能力

| 原生能力 | 原因 | 我们的替代方案 |
|----------|------|----------------|
| 原生投票机制 | 不满足3档分级需求 | 自主实现分层投票器 |
| 原生权限系统 | 不够精细 | 独立权限模块 |
| 原生工具注册 | 白名单机制不同 | MCP白名单层 |
| 原生Human-in-the-Loop | 确认流程不同 | 自主人工确认层 |

### 2.3 我们使用的OpenClaw能力

| 能力 | 如何使用 |
|------|----------|
| **Agent调度** | 使用其Agent生命周期管理 |
| **会话管理** | 使用其Session管理能力 |
| **工具调用协议** | 基于MCP协议封装 |
| **流式输出** | 使用其流式响应能力 |
| **日志框架** | 对接我们的审计系统 |

│  │  → L1: 主Agent投票 → 通过/否决                     │ │
│  │  → L2: 主+子Agent投票 → 2票通过/否决              │ │
│  └─────────────────────────────────────────────────────┘ │
│       │                                                    │
│       ▼                                                    │
│  ┌─────────────────────────────────────────────────────┐ │

│  │  → 无需确认 / 弱确认 / 强确认                       │ │
│  │  → 超时处理                                          │ │
│  └─────────────────────────────────────────────────────┘ │
│       │                                                    │
│       ▼                                                    │
│  输出：ExecutionPlan / Rejection                          │
│                                                             │
│  注意：完全自主实现，不依赖OpenClaw                       │
│                                                             │

## 4. 与OpenClaw的集成接口

### 4.1 集成架构

```
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw集成架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【我们自主开发的层】                                       │
│  ┌─────────────────────────────────────────────────────┐ │
│  │                                                       │ │
│  │   业务层  ←→  决策层  ←→  接入层                  │ │
│  │                                                       │ │
│  └─────────────────────────────────────────────────────┘ │
│                           │                                │
│                           │ 标准接口                       │
│                           ▼                                │
│  【OpenClaw内核层】                                      │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  SessionManager │ AgentRunner │ ToolRegistry        │ │
│  │  MessageBus │ StreamHandler │ AuditLogger           │ │
│  └─────────────────────────────────────────────────────┘ │
│                           │                                │
│                           ▼                                │
│  【模型层】                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 核心集成接口

| 接口 | 方向 | 说明 |
|------|------|------|
| **Agent调用** | 我们 → OpenClaw | 调用OpenClaw的Agent执行能力 |
| **Session管理** | 我们 → OpenClaw | 使用OpenClaw管理会话状态 |
| **工具注册** | 我们 → OpenClaw | 注册我们的MCP工具到OpenClaw |
| **日志输出** | OpenClaw → 我们 | 接收OpenClaw的操作日志 |
| **流式响应** | OpenClaw → 我们 | 接收Agent的流式输出 |
| **确认触发** | 我们 → OpenClaw | 通知OpenClaw暂停等待确认 |

### 4.3 不依赖的OpenClaw模块

| 模块 | 原因 | 我们的替代 |
|------|------|------------|
| OpenClaw的投票系统 | 不满足分层需求 | 自主实现 |
| OpenClaw的规则引擎 | 业务逻辑自主控制 | 独立规则引擎 |
| OpenClaw的权限系统 | 白名单机制不同 | 独立权限模块 |
| OpenClaw的确认流程 | UI逻辑自主控制 | 独立确认管理 |

## 5. OpenClaw使用策略

### 5.1 使用原则

```
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw使用原则                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  原则1：只使用经过验证的稳定能力                          │
│  → Agent调度、Session管理、流式输出                      │
│                                                             │
│  原则2：不依赖可能变更的内部实现                          │
│  → 使用官方公开API                                         │
│  → 封装隔离OpenClaw版本升级的影响                        │
│                                                             │
│  原则3：保持可替换性                                      │
│  → 定义清晰的接口抽象层                                    │
│  → 如有需要可替换为其他Agent框架                         │
│                                                             │
│  原则4：核心业务逻辑完全自主                              │
│  → 投票、风险、确认、审计全部自主实现                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 版本锁定策略

```markdown
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw版本管理                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 锁定版本号                                            │
│  → 不跟随OpenClaw大版本升级自动升级                       │
│  → 手动评估后决定是否升级                                 │
│                                                             │
│  2. 升级评估清单                                          │
│  → 是否有我们依赖的API变更                                │
│  → 是否有安全漏洞                                         │
│  → 是否有性能提升                                         │
│                                                             │
│  3. 升级流程                                              │
│  → 先在测试环境验证                                       │
│  → 确认接口兼容性                                         │
│  → 再发布到生产                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 7. 开发阶段规划

### 7.1 第一阶段：内核验证（1-2周）

```
目标：验证OpenClaw内核能力
任务：
1. 部署OpenClaw环境
2. 验证Agent调度能力
3. 验证Session管理能力
4. 验证工具调用能力
5. 输出：OpenClaw能力评估报告
```

### 7.2 第二阶段：核心模块开发（3-6周）

```
目标：开发自主核心模块
任务：
1. 开发分层投票决策器
2. 开发风险评估引擎
3. 开发白名单管理器
4. 开发人工确认工作流
5. 开发审计日志系统
```

### 7.3 第三阶段：集成与联调（7-8周）

```
目标：与OpenClaw集成
任务：
1. 设计与OpenClaw的接口适配层
2. 集成测试
3. 全流程联调
4. 性能测试
```

### 7.4 第四阶段：业务层开发（9-12周）

```
目标：开发业务规则和功能
任务：
1. 开发业务规则引擎
2. 开发MCP白名单工具
3. 开发前端UI
4. 客户内测
```

## 8. 风险与应对

### 8.1 主要风险

| 风险 | 影响 | 应对 |
|------|------|------|
| OpenClaw版本升级破坏兼容 | 高 | 锁定版本，接口抽象层隔离 |
| OpenClaw文档不足 | 中 | 深入源码，参考Claude Code实现 |
| 自主模块与OpenClaw集成困难 | 中 | 预留足够联调时间 |
| 性能不达标 | 中 | 分层投票减少API调用 |

### 8.2 可替换性保障

```
如果未来OpenClaw不再满足需求：

替换方案1：切换到LangChain
替换方案2：切换到LlamaIndex
替换方案3：完全自研Agent内核

由于我们的核心逻辑完全不依赖OpenClaw，
只需重写IAgentKernel接口实现即可。
```
