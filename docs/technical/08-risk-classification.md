# 风险分级体系

> 本文档定义3档风险分级标准及对应的投票机制，实现安全与效率的平衡。

---

## 1. 分级设计原则

### 1.1 为什么要3档而不是5档

| 对比 | 5档方案 | 3档方案 |
|------|---------|---------|
| 精细度 | 更精细 | 足够覆盖 |
| 配置复杂度 | 高 | 低 |
| 用户理解难度 | 高（难以判断自己属于哪档） | 低（容易理解） |
| 运营维护成本 | 高 | 低 |

### 1.2 3档设计思路

```
┌─────────────────────────────────────────────────────────────┐
│                    3档分级设计                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【自动车道】L0：查询类                                    │
│  → 主Agent直接返回 → 无投票                                │
│  → 用户感受：秒级响应                                      │
│                                                             │
│  【普通车道】L1：普通操作（<1万）                         │
│  → 主Agent + 子Agent1 → 2票通过                           │
│  → 用户感受：1-2秒响应                                    │
│                                                             │
│  【审核车道】L2：高风险操作（>1万）                        │
│  → 主Agent + 子Agent1 + 子Agent2 → 3票全通过              │
│  → 用户感受：需要确认，3-5秒                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 分级判断矩阵

### 2.1 四维判断体系

判断一个操作属于哪个等级，需要同时考虑4个维度：

```
┌─────────────────────────────────────────────────────────────┐
│                    四维判断体系                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  维度1：操作类型（最高权重）                                │
│  维度2：金额大小                                            │
│  维度3：操作字段敏感度                                      │
│  维度4：客户/上下文风险                                     │
│                                                             │
│  最终等级 = max(维度1, 维度2, 维度3, 维度4)             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 维度1：操作类型分级

| 操作类型 | 子类型 | 基础等级 | 说明 |
|----------|--------|----------|------|
| **query** | - | L0 | 只读，无任何风险 |
| **create** | 创建客户 | L1 | 新增数据，可逆 |
| **create** | 创建订单 | L1 | 涉及库存和金额 |
| **create** | 创建报价 | L1 | 商务操作，可取消 |
| **create** | 创建收付款 | L1 | 财务操作，可逆 |
| **update** | 修改客户信息 | L1 | 非核心字段 |
| **update** | 修改订单状态 | L1→L2 | 取决于金额 |
| **update** | 修改价格/折扣 | L2 | 涉及金额，需确认 |
| **update** | 修改库存 | L1→L2 | 取决于数量 |
| **update** | 修改信用额度 | L2 | 高风险 |
| **delete** | 删除报价 | L1 | 未成交前 |
| **delete** | 取消订单 | L2 | 涉及履约 |
| **delete** | 删除客户 | L2 | 涉及历史数据 |
| **delete** | 删除库存记录 | L2 | 不可逆 |
| **delete** | 清空操作 | L3 | 极高风险 |

### 2.3 维度2：金额分级

| 金额范围 | 基础等级 | 调整规则 |
|----------|----------|----------|
| **¥0** | L0 | 查询类，无金额 |
| **¥0 < 金额 ≤ ¥1,000** | L1 | 小额操作 |
| **¥1,000 < 金额 ≤ ¥10,000** | L1 | 普通操作 |
| **¥10,000 < 金额 ≤ ¥100,000** | L2 | 中额操作 |
| **¥100,000 < 金额 ≤ ¥500,000** | L2 | 大额操作 |
| **金额 > ¥500,000** | L3 | 极高风险 |

### 2.4 维度3：操作字段敏感度

| 敏感等级 | 字段类型 | 风险说明 |
|----------|----------|----------|
| **S0** | id, created_at, updated_at | 系统字段，无风险 |
| **S1** | name, phone, address | 基本信息，低风险 |
| **S2** | email, bank_account | 联系/财务信息，中风险 |
| **S3** | price, cost, discount | 价格字段，高风险 |
| **S4** | credit_limit, credit_line | 信用额度，极高风险 |

### 2.5 维度4：客户/上下文风险

| 客户类型 | 风险系数 | 说明 |
|----------|----------|------|
| **普通客户** | 1.0 | 标准风险 |
| **新客户** | 1.5 | 首次交易，不确定性强 |
| **VIP客户** | 0.8 | 高价值客户，低风险 |
| **高风险客户** | 2.0 | 信用不佳或被标记 |

## 3. 精确分级标准

### 3.1 L0 查询类操作（无需投票）

#### 3.1.1 L0 判定条件（必须全部满足）

```
┌─────────────────────────────────────────────────────────────┐
│                    L0 查询类判定条件                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  条件1：操作类型 = query                                   │
│  条件2：金额 = ¥0                                          │
│  条件3：不涉及S3/S4敏感字段                                │
│  条件4：结果对业务无变更影响                                │
│                                                             │
│  必须同时满足4个条件，才属于L0                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.2 L0 操作示例

| 操作 | 场景 | 是否L0 | 原因 |
|------|------|---------|------|
| 查询客户列表 | 按名称筛选 | ✅ | query + 无金额 |
| 查询客户详情 | 查看基本信息 | ✅ | query + 无金额 |
| 查询订单列表 | 按状态筛选 | ✅ | query + 无金额 |
| 查询订单详情 | 查看订单内容 | ✅ | query + 无金额 |
| 查询库存数量 | 查看SKU库存 | ✅ | query + 无金额 |
| 查询应收款 | 查看金额 | ✅ | query + 无金额 |
| 导出客户数据 | 批量导出 | ⚠️ | 可能涉及数据量，需评估 |
| 查询系统日志 | 查看操作记录 | ✅ | query + 无金额 |

### 3.2 L1 普通操作（主Agent通过）

#### 3.2.1 L1 判定条件（满足任一即可）

```
┌─────────────────────────────────────────────────────────────┐
│                    L1 普通操作判定条件                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  条件1：操作类型 in [create] + 金额 ≤ ¥10,000            │
│  条件2：操作类型 in [update] + 金额 ≤ ¥10,000 + 仅涉及S0-S2 │
│  条件3：金额 in (¥1,000, ¥10,000] + 操作类型为create      │
│                                                             │
│  满足任一条件，即属于L1                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.2.2 L1 操作示例

| 操作 | 场景 | 等级 | 判定原因 |
|------|------|------|----------|
| 创建客户 | 新增客户信息 | L1 | create + 金额0 |
| 创建订单 | 订单金额¥5,000 | L1 | create + 金额≤1万 |
| 创建报价 | 报价单金额¥8,000 | L1 | create + 金额≤1万 |
| 创建收付款记录 | 收款¥2,000 | L1 | create + 金额≤1万 |
| 修改客户电话 | 更新联系方式 | L1 | update + 仅S1字段 |
| 修改客户地址 | 更新地址信息 | L1 | update + 仅S1字段 |
| 修改订单备注 | 添加备注信息 | L1 | update + 仅S0-S1字段 |
| 修改收付款状态 | 标记已收款 | L1 | update + S0-S1字段 |
| 取消报价 | 取消未成交报价 | L1 | 操作可逆+金额≤1万 |

#### 3.2.3 L1 否定清单（不属于L1的操作）

| 操作 | 场景 | 否定原因 |
|------|------|----------|
| 创建订单 | 金额>¥10,000 | 金额超阈值 |
| 创建订单 | 金额>¥1,000 + 涉及S3字段 | 涉及价格字段 |
| 修改客户信用额度 | 任何金额 | 涉及S4字段 |
| 修改订单价格 | 任何金额 | 涉及S3字段 |
| 删除客户 | 任何金额 | delete操作 |
| 删除订单 | 任何金额 | delete操作 |

### 3.3 L2 重要/高风险操作（主+子Agent通过）

#### 3.3.1 L2 判定条件（满足任一即可）

```
┌─────────────────────────────────────────────────────────────┐
│                    L2 重要操作判定条件                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  条件1：金额 > ¥10,000                                     │
│  条件2：涉及S3敏感字段（price, cost, discount）           │
│  条件3：涉及S4敏感字段（credit_limit, credit_line）       │
│  条件4：操作类型 = delete                                  │
│  条件5：批量操作（>10条）                                   │
│  条件6：涉及金额调整（降价、折扣修改）                       │
│                                                             │
│  满足任一条件，即属于L2                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.3.2 L2 操作示例

| 操作 | 场景 | 等级 | 判定原因 |
|------|------|------|----------|
| 创建订单 | 订单金额¥50,000 | L2 | 金额>10万 |
| 创建订单 | 订单金额¥150,000 | L2 | 金额>10万 |
| 修改订单价格 | 折扣改为8折 | L2 | 涉及S3字段 |
| 修改订单状态 | 已确认→已完成 | L2 | 涉及履约状态 |
| 修改信用额度 | 调整客户额度 | L2 | 涉及S4字段 |
| 修改客户价格权限 | 允许自定义价格 | L2 | 高风险权限 |
| 取消订单 | 取消已确认订单 | L2 | delete+已履约 |
| 删除客户 | 删除客户档案 | L2 | delete+历史数据 |
| 批量修改库存 | 修改>10个SKU | L2 | 批量操作 |
| 批量调整价格 | 统一上调10% | L2 | 批量+涉及S3 |
| 删除报价 | 删除已发报价 | L2 | delete+已发出 |
| 创建收付款 | 收款>¥10,000 | L2 | 金额>10万 |

#### 3.3.3 L2 强制确认场景

以下L2操作必须额外验证：

| 操作 | 额外验证项 |
|------|------------|
| 取消订单 | 必须确认库存已释放 |
| 删除客户 | 必须确认无进行中订单 |
| 修改信用额度 | 必须显示调整前后对比 |
| 大额创建订单（>50万） | 必须二次密码验证 |

## 4. 复合场景分级

### 4.1 多维度叠加规则

```
当一个操作同时涉及多个维度时，按最高等级判定：

示例1：
- 操作类型：create（基础L1）
- 金额：¥50,000（触发L2）
- 结果：L2（取较高等级）

示例2：
- 操作类型：update（基础L1）
- 金额：¥5,000（属于L1）
- 涉及字段：price（S3敏感字段）
- 结果：L2（敏感字段升级）

示例3：
- 操作类型：create（基础L1）
- 金额：¥8,000（属于L1）
- 客户类型：高风险客户（风险系数2.0）
- 结果：L2（客户风险升级）
```

### 4.2 典型复合场景分级

| 场景 | 操作 | 金额 | 字段 | 客户 | 最终等级 | 判定原因 |
|------|------|------|------|------|----------|----------|
| 创建订单 | create | ¥5,000 | - | 普通 | L1 | 金额+类型 |
| 创建订单 | create | ¥5,000 | - | 高风险 | L2 | 客户升级 |
| 创建订单 | create | ¥50,000 | - | 普通 | L2 | 金额升级 |
| 修改价格 | update | ¥0 | price | 普通 | L2 | S3字段 |
| 修改电话 | update | ¥0 | phone | 普通 | L1 | S1字段 |
| 修改额度 | update | ¥0 | credit_limit | 普通 | L2 | S4字段 |
| 取消订单 | delete | ¥80,000 | - | 普通 | L2 | delete+金额 |
| 删除客户 | delete | ¥0 | - | 普通 | L2 | delete操作 |

| T0 | ¥0 | 无金额，纯查询 |

| T2 | ¥10,000 | 普通/重要边界 |
| T3 | ¥100,000 | 重要/大额边界 |
| T4 | ¥500,000 | 大额/极高边界 |

### 5.2 金额阈值动态调整

```
┌─────────────────────────────────────────────────────────────┐
│                    动态金额阈值                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  客户月交易额 > ¥500,000：                                 │
│  → 所有阈值提升50%                                        │
│  → T2从¥10,000变为¥15,000                                │
│                                                             │

│  → 所有阈值提升100%                                        │
│  → T2从¥10,000变为¥20,000                                │
│                                                             │
│  新客户（首次交易）：                                       │
│  → 所有阈值降低50%                                        │
│  → T2从¥10,000变为¥5,000                                 │
│                                                             │
│  高风险客户：                                               │
│  → 所有阈值降低50%                                        │
│  → T2从¥10,000变为¥5,000                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 7. 客户类型分级

### 7.1 客户分类标准

| 客户类型 | 分类标准 | 风险系数 | 阈值调整 |
|----------|----------|----------|----------|
| **普通客户** | 完成3次以上交易 + 信用正常 | 1.0 | 无调整 |
| **新客户** | 首次交易或交易<3次 | 1.5 | 阈值减半 |
| **VIP客户** | 月交易>10万 + 信用优秀 | 0.8 | 阈值提升50% |
| **高风险客户** | 逾期付款>3次或被投诉 | 2.0 | 阈值减半+强制确认 |
| **受限客户** | 被系统标记为受限 | 3.0 | 所有操作L2+ |

### 7.2 客户历史权重

```
┌─────────────────────────────────────────────────────────────┐
│                    客户历史权重                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  加分项（降低风险）：                                        │
│  ├── 交易次数 > 10次 → -0.2                              │
│  ├── 交易金额 > ¥500,000 → -0.2                           │
│  ├── 连续6个月无否决 → -0.3                               │
│  └── VIP标识 → -0.3                                      │
│                                                             │
│  减分项（增加风险）：                                        │
│  ├── 过去30天有否决 → +0.3                                │
│  ├── 逾期付款记录 > 1次 → +0.5                            │
│  ├── 客户类型变更（升级/降级）→ +0.2                      │
│  └── 被投诉记录 → +0.5                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 9. 完整分级决策树

### 9.1 决策流程

```
开始分级判断
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 1：判断操作类型                        │
│  query → L0（结束）                        │
│  create/update/delete → Step 2              │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 2：判断金额                            │
│  金额 = 0 → Step 3                        │
│  金额 > 0 → Step 4                        │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 3：分析操作字段                        │
│  涉及S3/S4 → L2（结束）                   │
│  仅涉及S0-S2 → Step 5                    │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 4：金额判断                            │
│  金额 ≤ ¥10,000 → L1（结束）              │
│  金额 > ¥10,000 → L2（结束）             │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 5：客户风险调整                        │
│  高风险客户 → L2（结束）                   │
│  普通客户 → L1（结束）                     │
└─────────────────────────────────────────────┘
```

## 10. 监控与告警

### 10.1 分级分布监控

| 指标 | 正常范围 | 告警阈值 | 处理 |
|------|----------|----------|------|
| L0占比 | 50-60% | <40% 或 >70% | 检查分级是否准确 |
| L1占比 | 30-40% | <20% 或 >50% | 检查分级是否准确 |
| L2占比 | 10-20% | <5% 或 >30% | 检查分级是否准确 |
| L0成功率 | >99% | <98% | 检查Agent状态 |
| L1成功率 | >95% | <90% | 检查否决原因 |
| L2成功率 | >90% | <80% | 检查确认率 |

## 11. 分级配置表（可直接配置）

### 11.1 操作类型配置

```yaml
# 风险分级配置
risk_levels:
  L0:
    operations:
      - query
    max_amount: 0
    allowed_fields: [S0, S1, S2]

  L1:
    operations:
      - create: [customer, quote, payment]
      - update: [customer_info, order_remark, status]
      - delete: [quote]
    max_amount: 10000
    allowed_fields: [S0, S1, S2]
    conditions:
      - no_S3_fields
      - no_S4_fields

  L2:
    operations:
      - create: [order, payment]  # 金额>1万
      - update: [price, discount, credit_limit]
      - delete: [customer, order]
    min_amount: 10001
    forced_fields: [S3, S4]
    conditions:
      - batch_operation
      - amount_over_threshold
```

### 11.2 阈值配置

```yaml
# 金额阈值配置
thresholds:
  T0_amount: 0
  T1_amount: 1000
  T2_amount: 10000
  T3_amount: 100000
  T4_amount: 500000

  # 客户类型调整
  customer_adjustments:
    normal: 1.0
    new: 0.5
    vip: 1.5
    high_risk: 0.5
    restricted: 0.25
```
